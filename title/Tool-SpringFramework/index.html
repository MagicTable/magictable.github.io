简介

  • SpringMVC是一种基于Java实现MVC模型的轻量级Web框架。

特点

  • 简化开发：降低企业级开发的复杂度。
  • 框架整合：高效整合其他技术，提高企业级应用开发与运行效率。

Spring框架设计思想
使用对象时不仅可以直接从IoC容器中获取bean对象，并且DI绑定好了所需的依赖关系。
通过xml配置文件的方式来降低耦合度。
通过注解的方式来减少配置文件中的数量。

Spring Framework
Spring Boot
Spring Cloud（分布式开发）

IoC（Inversion of Control）控制反转

  • 控制反转，对象的创建控制权由程序外部提供。
  • Spring提供了一个容器，称为IoC容器，用来充当IoC思想中的“外部”。
  • IoC容器负责对象的创建，初始化等一些列工作。
  • IoC容器中被管理的对象被称为Bean。

DI（Dependency Injection）依赖注入

  • 在容器中建立Bean与Bean之间依赖关系的过程，被称为依赖注入。

xml配置文件

beans标签

  • Spring最外层的标签。

bean标签

  • 功能：定义Spring容器管理的对象。

格式

属性

  • id：对象的唯一id，容器可以通过id获取对应的bean对象。
  • class：对象的位置，配置bean的全路径类名。
  • name：对象的别名，允许同时添加多个，允许使用逗号,、分号;、空格 分割。
  • scope：对象是否单例，参数（singleton单例<默认>、prototype非单例）。

案例

property标签

  • 依赖注入
  • <property>标签中的name参数是指定当前bean对象的class类中的方法名。
  • <property>标签中的ref参数是所依赖的bean对象的name别名。

Spring的注解

@Component
@Value（注入基本数据类型）
@Autowired（注入引用数据类型）
控制层：@Controller（控制器）、@RequestMapping(<”请求模块”>)（路径映射，请求，支持所有请求格式）、@GetMapping()、@ResponseBody（响应）
业务层：@Service
持久层：@Repository
@Bean
@Inport

接收传递过来的参数、参数名映射
普通风格：@RequestParam(“<参数>”)
REST full风格：@PathVariable(“<参数>”)
post请求body中的JSON数据参数：@RequestBody

为了解决的问题：传递的参数名不同时通过 参数名映射注解进行解耦，但不能动态修改。
