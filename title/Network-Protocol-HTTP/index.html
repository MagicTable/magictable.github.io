Network | 协议 - HTTP（超文本传输协议）

文章摘要： 在网络模型中属于应用层的网络通讯协议，HTTP全称：超文本传输协议。

简介

  • URI：统一资源标识符，代表网络中某个资源的名称。
  • URL：统一资源定位符，网址，代表网络设备的地址。
  • URN：

例子：http://www.baidu.com:80/index.html?name=xiaoming&age=18#a

  • 协议：http
  • IP/域名：www.baidu.com
  • 端口：80
  • 请求资源：/index.html
  • 参数：name=xiaoming&age=18
  • 锚点：a

客户端和服务端交互
客户端发送请求：request
服务端返回响应：response
通过URL来指定要获取的资源
响应内容是HTML网页，或者用json表示的数据或者其他二进制文件内容。

web使用一种名为http的协议作为规范，完成从客户端到服务端的一系列运行流程，而协议是指定规范的约定，web是建立在http协议上的通信。

作用：

  • 传输文本、音频、视频、文件等。
  • 专门用来两点传输内容的协议，包括请求方和响应方
  • 规定交互的约定和规范

缺点

  • HTTP基于DNS、IP、TCP协议
  • HTTP是一个公开协议、任何人都可以随时了解协议细节、模仿、伪造HTTP请求获取服务器端资源。
  • HTTP1.1版本歇息本身没有加密规则，因此整个报文都是明文在网络上流转。
  • HTTP是为网络数据传输而生，网络上任何一个传输节点、代理、网管、路由器等，都可以随时知道HTTP传输的内容。

URL统一资源定位符

  • 协议名、服务器地址、服务器端口号、带层次的资源路径、请求附带的参数、片段标识
