网络通讯基础知识

网络通讯架构

CS架构（Client客户端/Server服务端）

优点

  • 界面和操作可以很丰富。
  • 大部分数据保存在客户端，相对安全。
  • 大部分功能都集成在客户端，只需从服务器下载少量数据，因此访问数据快。

缺点

  • 升级维护工作量较大，每一个客户端都需要升级。
  • 用户群体固定，程序需要安装才能使用。

BS架构（Browser浏览器/Ssrver服务端）

优点

  • 客户端无需安装，有web浏览器即可。
  • 无需升级多个客户端，升级服务器即可。
  • BS架构可以直接放在广域网上，通过一定的权限控制实现多客户端访问的目的，交互性较强。

缺点

  • 在跨浏览器上，BS架构不尽如人意。
  • 表现要比CS架构的程序需要花费不少经历。
  • 在速度和安全性上需要花费巨大的设计成本。

网络通讯三要素

  • IP：设备在网络中唯一标识的地址。
  • 端口：应用程序在设备中唯一的标识。
  • 协议：连接和数据在网络中传输的规则。

IP

  • IP（Internet Protocol），全称“互联网协议地址”，是分配给联网设备在网络中唯一标识的地址。
  • IP地址有两种形式：IPv4、IPv6。
  • IPv4 由32位二进制组成，并将每8位二进制数转换为十进制数，使用点号.分开，例如：192.168.0.1
  • IPv6 由128位二进制组成，分成8段表示，每段每四位编码成一个十六进制位标识，数之间用冒号:分开，例如：2001:0db8:0000:2200:0008:0800:110a:417a

IP域名

  • 流程是输入域名发送到DNS服务器，查询域名对应的IP，再访问IP对应的服务器。例如：域名https://www.baidu.com，IP地址202.108.22.5

公网IP

  • 可以连接互联网的IP地址。

内网IP

  • 又称局域网IP，只能在同一网络下进行使用。

特殊IP地址

  • 127.0.0.1、localhost：代表本机IP，只会寻找当前所在的主机。

端口

  • 标记正在计算机设备上运行的应用程序。
  • 被规定为一个16位的二进制数，范围是0~65535

分类

  • 周知端口：0~1023，即被预先定义的知名应用程序占用。（如：HTTP占用80、FTP占用21）
  • 注册端口：1024~49151，即分配给用户进程或某些应用程序。
  • 动态端口：49152~65535，动态端口一般分配给不固定分配某些进程。

提示

  • 设备中不能出现两个程序的端口号重复。
  • 个人开发者开发的程序一般选择使用注册端口。

协议

  • 网络上通信的设备，事先规定连接规则、传输数据的规则被称为网络通讯协议。
  • OSI网络参考模型：全球网络互联标准。
  • TCP/IP网络模型：实际使用的国际标准。

UDP（用户数据报协议）

UDP（User Datagram Protocol）：用户数据报协议

  • 特点：无连接、不可靠通信。
  • 流程：不事先建立连接，数据按照包发，一包数据包含（自己的IP、程序端口、目的地IP、程序端口、数据）。
  • 优点：通信效率高。
  • 缺点：发送方不管对方是否在线，数据在中间丢失也不管，如果接收方收到数据也不返回确认。
  • 适用场景：语音通话、视频直播。

TCP（传输控制协议）

TCP（Transmission Control Protocol）：传输控制协议

  • 特点：面向连接、可靠通信，最终目的是要保证在不可靠的信道上实现可靠的传输。
  • 流程：
    1. 三次握手建立可靠连接（客户端发送连接请求、服务端返回响应、客户端再次确认信息建立连接）
    2. 传输数据进行确认
    3. 四次挥手断开连接（客户端发送断开连接请求、服务端返回稍等响应、服务端返回确认断开响应、客户端发送正式确认断开连接）
  • 优点：数据传输可靠。
  • 缺点：通信效率相对不高。
  • 适用场景：网页、文件下载、支付。
