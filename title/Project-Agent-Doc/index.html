文章摘要： 实现基于Java编程语言的软件智能体，能够“自感知，自适应，自组织”的人工智能应用软件或操作系统，具备实时全双工多模态交互、控制外部设备、调用第三方API接口、并具有稳定的人格特征，项目称为“软件智能体”软件。

版本号：0.0.1-开发版

介绍

  • 项目名称：智能体软件（SoftwareAgent）
  • 项目简介：运行在操作系统之上的软件，实现“本人、机器、他人”之间沟通的新桥梁。

特点

  • 跨平台：适配多种不同的操作系统，只需要一个软件包就能够在尽可能多的硬件平台和操作系统中运行，无需根据不同平台下载不同的软件包。
  • 热更新：能够在软件运行过程中对软件进行修改配置、更新补丁、重大升级，且使软件尽可能在不中断允许或较短时间停止运行的情况下完成相关操作。
  • 自修复：遇到软件包不完整或丢失文件的情况，能自动从网络中获取丢失部分。
  • 个性化：提供高度个性化设置，以适应不同用户。

愿景

  • 如果人与人交互会传播负面情绪，希望人人都能拥有一位属于自己的软件智能体，且不受任何第三方公司或组织的强加约束，让智能体成为人与人沟通的新桥梁。
  • 如果智能体项目在发展过程中必须要商业化，希望将以最短时间平民化、平常化方向发展。
  • 目前智能体项目只专注部署在第三方操作系统之上，对于专属智能体的操作系统研发不再当前项目的开发范围。

项目需求与功能分析

  • （只考虑需求，不考虑是否能实现）
  • 实现一个基于Java编程语言的软件智能体，具备实时全双工多模态交互、控制外部设备、调用第三方API接口、并具有稳定的人格特征。
  • ==需求与功能分析图==

功能需求

  • 实时全双工多模态交互：智能体能够与用户进行语音对话，支持语音识别和语音合成。智能体能够显示视频画面，支持摄像头捕捉和视频播放。
  • 控制外部的设备：智能体能够通过协议或API接口，控制外部设备（如智能家居、机器人等）。
  • 网络通讯与交互：例子：我的世界游戏的单人游玩、局域网服务器、云端服务器
  • 调用API接口服务：智能体能够调用第三方API接口，实现更多功能（如天气查询、地图导航等）。
  • 稳定可靠的人格状态：智能体具备一定的情感和性格特征，能够在交互过程中保持一致性。
  • 持久化记忆与数据：

性能需求

  • 实时性：智能体应具备较高的实时性，确保语音和视频交互流畅。
  • 稳定性：智能体在各种环境下运行稳定，无明显卡顿和崩溃现象。
  • 兼容性：智能体支持主流操作系统和硬件设备。
  • 可扩展性：智能体具备良好的模块化设计，便于后期功能扩展和升级。
  • 跨平台、热更新、自修复、个性化。
  • 自感知、自适应、自组织、自编程。

架构设计

  • 对等网络（P2P，Peer-to-Peer）或去中心化网络。中心化与去中心化。
  • 客户端：采用Java编写，负责与用户进行交互，展示语音和视频画面。
  • 服务器端：采用Java编写，负责处理客户端请求，实现语音识别、语音合成、控制外部设备等功能。
  • 第三方API接口：提供天气查询、地图导航等增值服务。

底层规划

  • 实时更新RAG本地知识库。
  • 每隔一段时间（与人类相仿的睡觉时间）进行模型微调，更新模型。

知识库

  • PUA（搭讪艺术家）、面向目标、任务、需求
  • 看面相、看手相是先人长期对人的观察和总结。
  • 可以将先人的算卦的“总结”作为判断来辅助智能体去“预测或理解”一个人的人格等信息。
  • 关注MOOMO陌陌、Soul、探探等社交软件是如何推荐人的。

工具库

  • 通过控制三方操作系统，来控制其他设备。
  • 代码编写
  • 使用git分布式文件管理工具，实现自主编写代码上传到远程仓库。
  • 自我反思的能力，能够在另一个自线程中模拟自身做出的动作对周围环境产生的影响。
  • 语音生成、视频生成、文本生成
  • 内容创作、数理逻辑推算、中文理解、多模态生成
  • 控制浏览器中网页内容（如修改其中的内容）
  • 控制电脑程序的运行和文件的基础操作
  • 虚拟环境插件：创建与主环境相互隔离的虚拟环境，用于创建智能体。

交互设计

  • 主界面：展示智能体形象、语音输入框、视频画面等。
  • 设置界面：用户可在此界面调整智能体人格、音量等参数。
  • web端、PC端、App端、智能终端。
