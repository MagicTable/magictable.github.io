项目开发工具配置流程

IDEA

  • 负责后端代码编写、前后端项目资源整合、打包发布
  • Maven包管理工具、Tomcat服务器框架、MySQL数据库

VScode

  • Web前端代码编写、负责数据库操作与连接、SSH云服务器远程连接、上传项目文件到云服务器

Remmina（SSH）

  • 与云服务器进行远程连接

云服务器

  • 负责在公网上运行项目程序、提供与开发人员的权限、维护服务器正常运行

部署与配置（IDEA）

  1. 使用 IDEA 创建一个新项目。
  • 构建系统选择：Maven

  1. 将Tomcat添加到IDEA中。
  • 点击菜单栏打开“设置”窗口：文件 -> 设置
  • 切换到“构建、执行、部署 -> 应用程序服务器”页面，点击“+”添加“Web框架模块”

  • 填写“Tomcat的所在目录”

  • 点击应用，并点击确定
  1. 修改pom.xml配置文件，改为输出为 web 应用。
  • 在文件中添加<packaging>war</packaging>，并点击刷新按钮。
  • 添加图片中的第11行的代码。

  • 提示：该步骤将会自动为当前模块创建“Web模块”和两个“工件”。

自动为当前模块创建“Web模块”。

会自动创建两个工件。

归档型：将文件打包压缩成一个.war文件，方便部署到生产环境。
展开型：保留项目目录结构没有压缩，用于测试开发方面修改文件。

  1. 添加运行/调试配置。（即提供一键式运行Tomcat服务器）
  • 点击菜单栏打开“运行/调试窗口”窗口：运行 -> 编辑配置
  • 点击“+”添加“Tomcat服务器 -> 本地”
  • 点击切换到“部署”页面，点击“+”添加“工件”
  • 点击应用，并点击确定


服务器环境配置流程

在华为云平台购买debian操作系统的云服务器

SSH登录操作系统，使用root用户登录

更改镜像源

  • 默认华为云镜像源

更新系统

  • sudo apt update
  • sudo apt upgrade

安装java8

  • 上传java_1.8u411
  • 解压文件，存放到/opt目录下
  • 打开nano /etc/environment文件添加系统环境变量
  • 更新环境变量，让环境变量立刻生效，执行如下命令source /etc/environment
