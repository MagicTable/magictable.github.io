摘要： 实现能够“自感知，自适应，自组织”的人工智能应用软件或操作系统，项目称为“软件智能体”软件。

版本号：0.0.1-开发版

介绍

  • 项目名称：智能体软件（SoftwareAgent）
  • 项目简介：运行在操作系统之上的软件，实现“本人、机器、他人”之间沟通的新桥梁。

愿景

  • 如果人与人交互会传播负面情绪，希望人人都能拥有一位属于自己的软件智能体，且不受任何第三方公司或组织的强加约束，让智能体成为人与人沟通的新桥梁。
  • 如果智能体项目在发展过程中必须要商业化，希望将以最短时间平民化、平常化方向发展。
  • 目前智能体项目只专注部署在第三方操作系统之上，对于专属智能体的操作系统研发不再当前项目的开发范围。

项目发展规划

  • 记录项目开发日志
  • 编写项目官方文档：介绍文档、协议与许可证文档、项目文档、开发者文档、API文档、常见问题（FAQ）
  • 搭建项目所需硬件：官方服务器端部署项目，并长期运行。
  • 规划项目发布政策：安全规划、开源许可协议、项目依赖第三方程序规划、第三方公司合作协议、国家政策、地区政策。
  • 创建项目所需公司：建立公司并维持运作。
  • 搭建项目开源社区：智能体创作创意工坊平台网站。
  • 智能体项目将会拥有官方云服务平台，其服务器内部有一位智能体，该智能体将会管理云服务器平台与其他智能体和用户进行交互。

开发进度

  • 项目当前的核心逻辑仍在开发中，暂时不可用。

项目需求

  • 整理依赖尽可能没有也不需要其他依赖文件或软件包，若有则尽可能将源码完美整合进行程序。
  • 软件与数据分离的代码设计模式：在长期使用中，软件程序部分存储占用几乎不变，而使用数据会随使用过程中不断增加存储占用。因此需要做到对数据有较好的扩展、备份、迁移的能力。
  • 跨平台：适配多种不同的操作系统，只需要一个软件包就能够在尽可能多的硬件平台和操作系统中运行，无需根据不同平台下载不同的软件包。
  • 热更新：能够在软件运行过程中对软件进行修改配置、更新补丁、重大升级，且使软件尽可能在不中断允许或较短时间停止运行的情况下完成相关操作。
  • 自修复：遇到软件包不完整或丢失文件的情况，能自动从网络中获取丢失部分。
  • 个性化：提供高度个性化设置，以适应不同用户。
  • 自感知：能够自主接收声光电等多种形式、多种接口的输入。
  • 自适应：能够自行改变底层驱动，运行在各类底层环境上。
  • 自组织：能够自行组成系统，为各类硬件提供驱动，并为软件提供运行环境和资源调度。

适用于智能体的操作系统

  • 智能体项目本身就是一个操作系统，而运行在硬件上时需要通过对接管理硬件的BIOS程序来控制硬件。
  • 智能体项目运行的操作系统应当是一个但用户多线程实时的操作系统，专门以智能体为中心，系统本身自带命令终端界面也可选择性的安装图像界面，也可以通过专门与智能体交互的软件（软件可以安装在各类用户设备上）。

核心思想

软件智能体的文件存储布局

  • 软件智能体项目的“程序核心”、“程序文档”、“智能体个体数据”、“图形交互终端软件”之间的文件相互分开存储。（将静态文件与动态文件分离存储提高数据安全性和读取效率，也可以灵活更换另一个智能体的个体数据，但“智能体个体数据”需要与“程序核心”的版本一致）
  • “程序”部分由四个部分组成：核心代码、工具文件、配置文件、日志文件、数据文件、环境依赖文件。
  • “智能体”部分由七个部分组成：心智、形象、记忆、配置文件、日志文件、工具、缓存数据。

项目程序架构的设计

  • 强调可扩展性和可维护性，以确保软件能适应未来的需求变化和升级。
  • 提出“manager”管理器的概念，将核心代码整理到该目录下，对于后续扩展代码将存储在“tool”工具目录中。
  • 通过不改变管理器下的代码，更改工具中的代码。

智能体项目的运行方式

  • 智能体项目是运行在操作系统中的一个软件，同时只能运行一个独立个体，但可以通过虚拟环境工具，创建由“独立个体”控制的“虚拟个体”。
