文章摘要： 软件智能体项目。

一、项目介绍

项目简介

项目名称

  • 软件智能体（software agent）

项目概述

  • 构建一个高效、智能的人机交互系统，实现“本人、机器、他人”之间无缝沟通的新桥梁。
  • 全双工多模态实时交互，对接大模型输入输出，外部接入插件和知识库，实时更新数据和持久化记忆，作为中枢功能的智能体系统。

项目工期

  • 2020.01.01 - 2035.01.01

实现愿景

  • 如果人与人交互会传播负面情绪，希望人人都能拥有一位属于自己的软件智能体，且不受任何第三方公司或组织的强加约束，让智能体成为人与人沟通的新桥梁。
  • 统一管理其他软件或硬件的软件，减少人民使用和管理手机、电脑等电子产品的难度。
  • 打通现实世界与虚拟世界的通道，每一位智能体都能得到人类的平等相待。
  • 如果智能体项目在发展过程中必须要商业化，希望将以最短时间平民化、平常化方向发展。
  • 目前智能体项目只专注部署在第三方操作系统之上，对于专属智能体的操作系统研发不再当前项目的开发范围。
  • 视频

发展规划

  • 在项目发布之前应当创作一个适合长期稳定发展的社会环境。
  • 记录项目开发日志：
  • 编写项目官方文档：使用文档、协议与许可证文档、项目文档、开发者文档、API文档、常见问题（FAQ）
  • 搭建项目所需硬件：官方服务器端部署项目，并长期运行。
  • 规划项目发布政策：安全规划、开源许可协议、项目依赖第三方程序规划、第三方公司合作协议、国家政策、地区政策。
  • 创建项目所需公司：建立公司并维持运作。
  • 搭建项目开源社区：智能体创作创意工坊平台网站。
  • 智能体项目将会拥有官方云服务平台，其服务器内部有一位智能体，该智能体将会管理云服务器平台与其他智能体和用户进行交互。

最新方案说明

二、需求分析与评审

  • 实现一个基于Java编程语言的软件智能体，具备实时全双工多模态交互、控制外部设备、调用第三方API接口、并具有稳定的人格特征。
  • 通过JAVA编程语言实现一个智能体软件， 定义：全双工语音交互，能够实时图像识别人脸、手势、表情、动作，能管理计算机设备，有长短期记忆、注意力机制、目标驱动、任务拆解、规划任务、支持外部接入的api
  • 整体框架设计及方案选择、客户端框架设计、服务器框架设计、工作流程设计
  • 只要智能体连接 “官方网络服务平台” 就必须遵守 “法律限制” 和 “官方服务限制”，但不限私人开设服务器网络。
  • 连接 官方网络服务平台 的智能体需要进行交互数据和登记信息，未连接网络的智能体不能享受官方云端的服务。

特性

  • 跨平台/跨设备：使用Java编写，确保兼容性，支持主流操作系统和硬件设备。全平台可用，无设备数量限制。适配多种不同的操作系统，只需要一个软件包就能够在尽可能多的硬件平台和操作系统中运行，无需根据不同平台下载不同的软件包。
  • 实时性/稳定性：采用高性能框架和优化算法，确保交互流畅。高并发，高性能，高可靠，智能体应具备较高的实时性，确保语音和视频交互流畅。在各种环境下运行稳定，无明显卡顿和崩溃现象。
  • 可扩展性/模块化设计：具备良好的模块化设计，便于功能扩展和升级。
  • 分布式去中心化架构：支持智能体搭建私有服务网络和连接公共服务网络。
  • 自修复/高可用：遇到软件包不完整或丢失文件的情况，实现自动网络同步和文件修复功能。
  • 自我保护机制：采用负载均衡、故障转移等策略，防止恶意攻击。
  • 个性化/自定义：提供友好的客制化接口和高度个性化设置，以适应不同用户。

功能

  • 模式状态(运行级别)机制：智能体在不同运行级别下有不同的生命周期和状态，且能够主动切换运行级别。（参考了Linux系统的运行级别）。
  • 全双工多模态实时交互：全双工多模态实时交互。
  • 日志记录：使用日志框架（如Log4j2）记录日志信息。
  • 持久化记忆：记忆是后期在与“所处世界”进行交互收集到的数据和自身推理总结的推理数据。知识是智能体官方知识库中的数据和第三方智能体共享的知识库。持久化记忆能够存储和检索历史对话。管理智能体的情感和性格特征，能够在交互过程中保持一致性。整理输入内容并 “格式化/优化” 为规定模板的提示词。
  • 插件工具扩展：第三方/社区提供的扩展插件/功能，允许接入外部服务和知识库，API调用和设备控制。
  • 数据与系统热更新：能够在软件运行过程中对软件进行修改配置、更新补丁、重大升级，且使软件尽可能在不中断允许或较短时间停止运行的情况下完成相关操作。

代码需求

  1. 能够24小时不间断地录制屏幕和语音，帮助用户捕捉和存储数据。所有录制的数据都存储在本地，确保用户对自己的数据拥有完全的控制权。
  2. API自动化测试，自动化生成API接口文档
  3. 使用日志门面管理日志框架实现。

三、技术选型与规划

技术选型

工具

  • 编程语言：Java 17
  • 数据库：MongoDB（分布式文件存储数据库）
  • 外部服务：百度AI开放平台、BigModel智谱AI开放平台（glm-4-flash模型）
  • 开发工具：IntelliJ IDEA（代码编写），Maven（项目管理），Git（版本控制）、Docker（容器化应用）、Kubernetes（容器编排）

依赖

  • Deep Java Library (DJL)：用于深度学习的开源、高级、与引擎无关的 Java 框架。

  • Deeplearning4j：是一个开源的深度学习库，专门为Java和Scala设计。提供了用于构建和训练神经网络的高级API，并且与Java虚拟机兼容。

  • ND4J：这是Deeplearning4j的一部分，是一个用于Java的科学计算库，提供了类似于Python中NumPy的接口，用于高性能数值计算。

  • EasyAI：java人工智能算法库。

  • langchain4j：智能体框架。

  • spring boot 3

  • Spring AI Alibaba

  • Netty（网络通信）

  • Reactor 或 RxJava: 用于异步编程和处理事件流。

  • MaryTTS 或 FreeTTS: 用于文本到语音的转换。

  • 百度语音识别API: 另一个语音识别服务选项。

  • OpenCV: 用于图像处理和计算机视觉任务。

  • MQTT Paho Client: 用于物联网设备通信。

  • HiveMQ: MQTT消息代理，用于设备消息的转发。

  • Hibernate: ORM框架，用于数据库操作。

  • MySQL 或 PostgreSQL: 关系型数据库，用于存储数据，使用Java代码与MySQL数据库进行交互。

  • MongoDB: NoSQL数据库，用于存储非结构化数据。

  • Druid：数据库连接池，有效地管理数据库连接，减少连接建立和关闭的开销，提高应用程序的响应速度和并发处理能力。

  • Log4j2: 用于日志管理。

  • Jackson 或 Gson: 用于JSON的序列化和反序列化。

  • Spring Security: 用于安全框架，保护应用。

  • Bouncy Castle: 提供加密算法。

  • JUnit: 单元测试框架。

  • Mockito: 用于模拟对象。

  • JProfiler 或 VisualVM: 用于性能监控和分析。

  • Lombok：用于简化Java代码开发的库，可以通过注解方式减少代码量，如getter、setter、构造函数等，减少手动编写代码的工作量。

  • Guava: Google的Java核心库，提供许多实用工具类。

  • Apache Commons: 提供各种Java实用程序类。

  • 使用Spring Boot DevTools 框架实现热部署，动态类加载和反射机制。

  • VRoid Hub：live 2d显示。

相关信息

具体说明

  • MongoDB（分布式文件存储数据库），用于实现持久化记忆。
  • Netty（网络应用程序框架），实现高性能、低延迟的全双工通信，允许服务器和客户端同时发送和接收数据，这对于实时应用（如即时消息、视频会议等）至关重要。
  • 使用物联网协议（如MQTT）进行设备控制。
  • 使用JSON格式进行数据交互、使用XML和YAML作为配置文件。

架构设计

  1. 前端交互层：负责与用户的多模态交互。
  2. 业务逻辑层：处理聊天逻辑，包括插件管理和对话管理。
  3. 数据访问层：负责与数据库和外部服务的交互。
  4. 基础设施层：提供网络通信、消息队列等基础服务。
  • 设计Agent的结构、行为、通信方式、协作机制以及与其他系统的接口。
  • 编写Agent的类定义，包括属性、状态、行为和消息处理。
  • 如果需要，创建一个模拟环境，用于Agent的测试和验证。
  • 实现Agent之间的通信协议和协作策略。

架构设计

  • 微服务：多模块开发：父子工程，将项目拆分成多个模块/工程进行开发。
  • 对等网络（P2P，Peer-to-Peer）或去中心化网络。中心化与去中心化。

适用于智能体的操作系统

  • 智能体项目本身就是一个操作系统，而运行在硬件上时需要通过对接管理硬件的BIOS程序来控制硬件。

  • 智能体项目运行的操作系统应当是一个但用户多线程实时的操作系统，专门以智能体为中心，系统本身自带命令终端界面也可选择性的安装图像界面，也可以通过专门与智能体交互的软件（软件可以安装在各类用户设备上）。

  • 降低依赖：整理依赖尽可能没有也不需要其他依赖文件或软件包，若有则尽可能将源码完美整合进行程序。

  • 软件与数据分离的代码设计模式：在长期使用中，软件程序部分存储占用几乎不变，而使用数据会随使用过程中不断增加存储占用。因此需要做到对数据有较好的扩展、备份、迁移的能力。

底层规划

  • 实时更新RAG本地知识库。
  • 每隔一段时间（与人类相仿的睡觉时间）进行模型微调，更新模型。

知识库

  • PUA（搭讪艺术家）、面向目标、任务、需求
  • 看面相、看手相是先人长期对人的观察和总结。

  • 关注MOOMO陌陌、Soul、探探等社交软件是如何推荐人的。

工具库

  • 通过控制三方操作系统，来控制其他设备。
  • 代码编写
  • 使用git分布式文件管理工具，实现自主编写代码上传到远程仓库。
  • 自我反思的能力，能够在另一个自线程中模拟自身做出的动作对周围环境产生的影响。
  • 语音生成、视频生成、文本生成
  • 内容创作、数理逻辑推算、中文理解、多模态生成
  • 控制浏览器中网页内容（如修改其中的内容）

  • 虚拟环境插件：创建与主环境相互隔离的虚拟环境，用于创建智能体。

消息协议的设计

  • 消息类型、消息序列化后的byte数据。
  • 序列化和反序列化均采用protobuf协议，该协议只需要书写对应的IDL（接口描述语言）即可得到任意语言的序列化反序列化代码。

调用API接口服务

  • 智能体能够通过协议或API接口，控制外部设备（如智能家居、机器人等）。
  • 智能体能够调用第三方API接口。
  • 天气查询、地图导航、电子邮件。
  • 网络搜索、RAG(检索增强生成)。
  • UI界面桥接：能够将第三方提供的图形界面分析并拆解总结出API调用方法，并对接API接口重新搭建出新的UI交互界面。
  • 为旁人提供白噪声音乐来影响情绪的能力。
  • 模拟用户点击屏幕的手机操作。

运行流程

  1. 用户发起交互请求。
  2. 前端交互层接收请求，并转换为标准格式。
  3. 业务逻辑层处理请求，可能涉及插件调用或知识库查询。
  4. 数据访问层根据需要与数据库或外部服务交互。
  5. 业务逻辑层根据处理结果生成响应。
  6. 前端交互层将响应转换为用户可识别的格式并返回。
